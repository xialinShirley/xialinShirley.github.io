    • 拷贝继承

    • 组合继承：通过调用父类构造，继承父类的属性并保留传参的优点，然后通过将父类实例作为子类原型，实现函数复用。

  3. 由于setState会导致组件重新渲染，因而可以通过重新渲染的方式来传入新的props从而达到修改显示形态的效果。

  • state vs props
  1. state的主要作用是用于组件保存、控制、修改自己的可变状态。state在组件内部初始化，可以被组件自身修改，而外部不能访问也不能修改。可以把state看做是一个局部的、只能被组件自身控制的数据源。state中状态可以通过this.setState方法进行更新。setState会导致组件的重新渲染。
  2. props的主要作用是让使用该组件的父组件可以传入参数来配置该组件。它是外部传进来的配置参数，组件内部无法控制也无法修改。除非外部组件主动传入新的props，否则组件的props永远保持不变。
  3. state是让组件控制自己的状态，props是让外部对组件自己进行配置。尽量少使用state，尽量多用props。
  4. React.js鼓励无状态组件。以前一个组件是通过继承Component来构建，一个子类就是一个组件，而用函数式的组件编写方法一个函数就是一个组件。
  • 渲染列表数据
  1. 如果往{}放一个数组，React.js会将数组里面一个个元素罗列并且渲染出来。
  2. React.js的高效依赖所谓的Virtual-DOM策略，通过在列表中加入唯一的key能帮助判断元素位置的交换，从而尽量复用元素内部的结构。（注意key不能直接在props中调用）
  • 实战部分-评论功能
  1. 在React.js当中必须要用setState才能更新组件的内容。
  2. 类似<input //>、<select //>、<textarea //>这些元素的value值被React.js所控制、渲染的组件，在React.js当中被称为受控组件。对于用户可输入的控件，一般都可以让它们称为受控组件。
  • 前端应用状态管理–状态提升
  1. 当某个状态被多个组件依赖或者影响的时候，就把该状态提升到这些组件的最近公共父组件中去管理，用props传递数据或者函数来管理这种依赖或者影响的行为。
  2. input标签中type除了text外还可以填number，toFixed(n)可以将number四舍五入到n位小数。
  • 挂载阶段的组件生命周期（一）

  1. React.js将组件渲染并且构造DOM元素然后塞入页面的过程称为组件的挂载。

  2. 挂载的时候，React.js会在组件的render之前调用componentWillMount，在DOM元素塞入页面之后调用componentDidMount。

  3. 在组件删除之前React.js会调用组件定义的componentWillUnmount。

  • 挂载阶段的组件生命周期（二）
  1. setState只能在已经挂载或者正在挂载的组件上调用，因为React.js重新构造和销毁组件来达到隐藏和显示的效果，故对于隐藏的组件不能调用。
  2. 一些组件启动的动作，包括Ajax数据的拉取操作、一些定时器的启动，可以放在componentWillMount里面进行
  3. componentWillUnmount的作用是在组件销毁的时候做例如清除该组件定时器和其他的数据清理的清场工作，不能用this.setState({})进行赋值，要调用另一个函数进行赋值。
  4. 有些组件的启动工作是依赖DOM的，例如动画的启动，而componentWillMount时组件还未挂载完成，因而将这些启动操作放在componentDidMount当中。
  5. await关键字仅在async function中有效，async函数执行时，如果遇到await就会先暂停执行，等到触发的异步操作完成后，恢复async函数的执行并返回解析值。
  • 更新阶段的组件生命周期
  1. 更新阶段指React.js重新渲染组件并把组件的变化应用到DOM元素上的过程，是一个组件的变化过程。更新阶段的组件生命周期有：
    • shouldComponentUpdate(nextProps,nextState)：通过这个方法控制组件是否重新渲染。如果返回false组件就不会重新渲染。这个生命周期在React.js性能优化上非常有用。
    • componentWillReceiveProps(nextProps)：组件从父组件接收到新的props之前调用。
    • componentWillUpdata( )：组件开始渲染之前调用。
    • componentDidUpdate( )：组件重新渲染并且把更改变更到真实的DOM以后调用。

《react小书》笔记(1-10)

  • 使用JSX描述UI信息
  1. 一个DOM元素包含三个信息：标签名，属性，子元素。

    • 不一定会渲染到浏览器的普通页面
    • 更新组件时操作js对象而不是DOM能减少浏览器重排从而极大优化性能。
  • 组件的render方法
  1. render方法返回必须要用一个外层的JSX元素把所有内容包裹起来，返回并列多个JSX元素是不合法的。
  2. div占据新的一行，span是行内元素。
  • 组件的组合、嵌套和组件树
  1. 自定义的组件都必须要用大写字母开头，普通的HTML标签都用小写字母开头。
  2. 理解组件树的概念对理解数据是如何在组件树内自上往下流动过程很重要。
  • 事件监听

  1. on*事件监听封装在React.js中，只能用在普通的HTML标签上，而不能用在组件标签上。

  2. 查看React.js中的事件https://reactjs.org/docs/events.html#supported-events，事件属性名都必须要用驼峰命名法。

  3. 一般在某个类的实例方法中this指的是这个实例本身，但在handlexxx的this是null或者undefined，因为React.js并不是通过对象方法的方式调用this.handlexxx，而是直接通过函数调用handlexxx。所以事件监听函数不能通过this获取到实例。

  4. bind模式不仅可以把事监听方法中的this绑定到当前组件实例上，还可以在渲染列表元素的时候，把列表元素传入事件监听函数当中。

  • 组件的state和setState

  1. 使用setState来告知React.js修改了组件的状态便于更新页面，只需要传入需要更新的部分，而不需要传入整个对象。

  2. 调用steState时state不会马上被修改，而是放进更新队列中，稍后才会从队列当中把新的状态提取出来合并到state中，再触发组件更新。因此使用前一次结果传入这个函数。

  3. React.js内部会把Javascript事件循环中的消息队列的同一个消息中的setState都进行合并以后再重新渲染组件，并不需要担心多次进行setState会带来性能问题。

ES6阮一峰教程学习笔记 （4）字符串的扩展

字符串的扩展

  • 字符的Unicode表示法
  1. ES6将码点放入大括号，即\u{xxxx}，就能正确解读该字符。
  2. 大括号表示法与四字节的UTF-16编码是等价的。
  • 字符串的遍历器接口

  1. ES6中字符串可以被for…of循环遍历。

  2. for…of可以识别0xFFFF的码点，传统的for循环无法识别这样的码点。

  • 直接输入U+2028和U+2029

  1. Javascipt字符串允许直接输入字符，以及输入字符的转义形式。

  2. Javascript规定有5个字符，不能在字符串里直接使用，只能使用转义形式。

    • U+005C：反斜杠（reverse solidus)
    • U+000D：回车（carriage return）
    • U+2028：行分隔符（line separator）
    • U+2029：段分隔符（paragraph separator）
    • U+000A：换行符（line feed）

  3. 但为了适应JSON模板字符串允许直接输入行/段分隔符，正则表达式依然不允许直接输入这两个字符。

  • JSON.stringfy()的改造

  1. 为确保返回的是合法的UTF-8字符，ES2019改变了JSON.stringfy()的行为，如果遇到0xD800到0xDFFF之间的单个码点或者不存在的配对形式，将返回转义字符串，留给应用自己决定下一步处理。

  • 模板字符串

  1. ES6的模板字符串用反引号(`)标识，可以当作普通字符串使用，也可以用来定义多行字符串，或者在字符串中嵌入变量。

  2. 如果在模板字符串中需要使用反引号，则前面要用反斜杠转义。

  3. 如果使用模板字符串表示多行字符串，所有的空格和缩进都会被保留在输出之中。通过trim()方法可以消除换行。

  4. 模板字符串中嵌入变量，需要将变量名写在${}之中。如果大括号中的值不是字符串，将按一般的规则转为字符串。

  • 标签模板

  1. 模板字符串跟在一个函数名后面，该函数将被调用来处理这个模板字符串。这被称为“标签模板”功能。其中“标签”指函数，紧跟在后面的模板字符串是它的参数。

  2. 对于有变量的模板字符串，先将模板字符串处理成多个参数，再调用函数。

  3. “标签模板”重要应用

    • 过滤HTML字符串，防止用户输入恶意内容。
    • 多语言转换，可以在Javascript语言中嵌入其他语言。

  4. 模板处理函数的第一个参数（模板字符串数组），还有一个raw属性，保存的是转义后 的原字符串。比如，strings.raw 数组会将\n为\\和n两个字符，而不是换行符。这是为了方便取得转义之前的原始模板而设计的。

  • 模板字符串的限制
  1. ES2018遇到不合法的字符串转义返回undefined，而不是报错，从raw属性上可以得到原始字符串。
  2. 对字符串转义的放松，只在标签模板解析字符串时生效，不是标签模板的场合依然会报错。
