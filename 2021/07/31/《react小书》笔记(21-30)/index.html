《react小书》笔记(21-30)

  • ref和React.js中的DOM操作
  1. React.js当中提供了ref属性来帮助获取已经挂载的元素的DOM节点，可以给某个ISX元素加上ref属性，但能不用ref就尽量不要用。
  2. 使用ref的场景例子：
    • 进入页面后自动focus到某个输入框，调用input.focus( )的DOM API
    • 动态获取某个DOM元素来做后续的动画
  3. 组件标签加上ref获取到的是该组件在React.js内部初始化的实例，但不常用。
  • props.children和容器类组件
  1. 使用自定义组件的时候，可以在其中嵌套JSX结构。嵌套的结构在组件内部都可以通过props.children获取到，这种组件编写方式在编写容器类型的组件当中非常有用。
  • dangerouslySetHTML和style属性

  1. 要实现动态渲染元素的innerHTML结构，需要给dangerouslySetInnerHTML传入一个对象，这个对象的__html属性值就相当于元素的innerHTML。复杂是因为设置innerHTML可能会导致跨站脚本攻击。

  2. 在React.js中需要把CSS属性变成一个对象再传给元素。用对象作为style方便动态设置元素的样式，可以用props或者state中的数据生成样式对象再传给元素，然后用setState就可以修改样式，非常灵活。

  • PropTypes和组件参数验证

  1. Javascript的灵活性体现在弱类型、高阶函数等语言特性上。

  2. 文件头部引入PropTypes并给组件类添加了类属性propTypes，规定传入的comment类型必须为object（对象）。通过Required关键字来强制组件某个参数必须传入。

  3. PropTypes提供了一系列的数据类型可以用来配置组件的参数，具体见：https://reactjs.org/docs/typechecking-with-proptypes.html.

  4. 组件参数验证在构建大型组件库时能迅速定位这种类型错误，使组件开发更为规范，另外也起到一个说明文档的作用。

  • 实战分析：评论功能（四）
  1. 用户名输入框失去焦点的事件onBlur与onFocus相对。
  2. 组件的私有方法都以_开头，所有事件监听的方法都用handle开头。把事件监听方法传给组件的时候，属性名都用on开头。
  3. 便于维护的组件内容编写顺序：
    • static开头的类属性，如defaultProps、propTypes。
    • 构造函数，constructor。
    • getter/setter
    • 组件生命周期。
    • _开头的私有方法。
    • 事件监听方法，handle*。
    • render*开头的方法。有时候render()方法里面的内容会分开到不同函数里面进行，这些函数都以render*开头。
    • render()方法。

  • 高阶组件(Higher-Order Components)
  1. 高阶组件是一个函数，它接受一个组件作为参数，返回一个新的组件，新组建会将传入的组件作为子组件。
  2. 高阶组件其实就是为了组件之间的代码复用，可以把组件之间可复用的代码、逻辑抽离到高阶组件当中。新的组件和传入的组件通过props传递信息。
  • React.js的context

  1. 首先，尽量避免使用context。某个组件只要往自己的context里面放了些状态，这个组件之下的所有子组件都直接访问这个状态而不需要通过中间组件的传递。

  2. 一个组件的context只有它的子组件能够访问，它的父组件是不能访问到的。

  5. context打破了组件和组件之间通过props传递数据的规范，极大地增强了组件之间的耦合性。一些第三方的前端应用状态管理库（Redux）就是充分利用了这种机制提供便利的状态管理服务。

  • 动手实现Redux(一)：优雅地修改共享状态
  1. Redux是一种架构模式，不关注使用什么库，而React-redux是把Redux这种架构模式和React.js结合起来的一个库，是Redux架构在React.js中的体现。
  2. 解决的矛盾是：“模块（组件）之间需要共享数据”和“数据可能被任意修改导致不可预料的结果”。
  3. 解决方案：提高数据修改的门槛，设置“中间人”函数，所有的数据修改必须通过它。
