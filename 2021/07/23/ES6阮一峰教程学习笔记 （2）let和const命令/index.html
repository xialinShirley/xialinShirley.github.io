ES6阮一峰教程学习笔记 （2）let和const命令

let 和const 命令

  • let命令

  1. for 循环设置循环变量的部分是父作用域，循环体内部是子作用域。同一作用域不可使用let重复声明同一个变量。

  2. var存在变量提升的情况，在未声明时值为undefined，而相同情况let则会报错。

  3. let声明的变量“绑定”这个区域，可能会导致“暂时性死区”(TDZ)的出现,也可能导致typeof报错（在变量没有声明时返回“undefined”）。TDZ的本质是：只要一进入当前作用域，所要使用的 变量就已经存在了，但是不可获取，只有等到声明变量的那一行代码出现，才可以获取和使用该变量。

  4. 不能在函数内部重新声明参数。

  • 块级作用域

  1. 考虑到环境差异较大，应该避免在块级作用域内声明函数。如果确实需要，应该写成函数表达式而不是函数声明语句。

  2. ES6的块状作用域必须有大括号，let和函数声明只能出现在当前作用域的顶层。

  • const命令

  1. const一旦声明变量就必须立即初始化，变量不提升，存在TDZ，只能声明后位置使用，不能重复声明。（类似let性质）

  2. const保证的是变量指向的那个内存地址所保存的数据不得改动，对于对象和数组等复合型数据进行常量声明时必须非常小心。

  3. ES6有6种声明对象的方法：var,function（ES5包含）,let,const,class,import。

  • 顶层对象的属性

  1. 顶层对象，在浏览器环境指的是window对象，在Node中指的是global对象。

  2. var命令和function命令声明的全局变量依旧是顶层对象的属性，let命令、const命令、class命令声明的全局变量不属于顶层对象的属性。
