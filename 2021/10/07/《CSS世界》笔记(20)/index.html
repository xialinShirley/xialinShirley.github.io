《CSS世界》笔记(20)

  1. outline表示元素的轮廓，语法和border属性非常类似，分宽度、类型和颜色，支持的关键字和属性值和border属性一模一样。两者表现也类似，都是给元素的外面画框。

  2. 不能在全局设置outline:0 none，会重置一些默认focus发光效果。

  3. outline是真正意义上的不占据任何空间的属性。应用：

    • 头像剪裁得到矩形镂空效果。

      使用一个大大的outline来实现周围半透明遮罩，超出的区域通过容器overflow:hidden隐藏。

    • 自动填满屏幕剩余空间的应用技巧。

      设置超大范围的outline属性，让底部内容元素绝对定位，同时以上边缘和左边缘为界进行裁剪。

  4. cursor支持的值：

    • auto:cursor默认值。auto表示光标形状根据内容类别浏览器自动进行处理。

    • default:系统默认光标形状。其中，链接为手形，按钮应该是默认形，通过hover时候的状态变化让用户知道这里是可点击的、可以有交互的。

    • none:可以让光标隐藏不见。

    • pointer:光标表现为一只伸出食指的手。

    • help:光标上带了个问号。

    • progress:表示进行中的意思。从语义上讲，适合loading处理。

    • wait:不使用。

    • text/vertical-text：文字（垂直）选中。

    • crosshair:十字光标。通常用在像素级的框选或点选场合。、

    • cell:Excel中单元格框选同款。

    • move:指示当前元素是可以移动的。

      太多了自己看323页开始的吧。

  5. 自定义光标：对于Chrome等浏览器，可以直接使用PNG图片作为光标，但是IE浏览器只支持专门的.cur格式的光标文件，需要使用工具进行格式转换。

  6. direction支持ltr(left-to-right,默认值)和rtl(right-to-left)。direction:rtl还可以让text-justify两端对齐元素，最后一行落单的元素右对齐显示。在不支持text-align:start/end 的浏览器中（如 IE），不同的 direction 属性值会改变 text-align 属性的初始值。

  7. unicode-bidi中embed和normal的区别是，embed 属性值的字符排序是独立内嵌的，不受外部影响。

  8. writing-mode 就是用来实现文字竖向呈现的。相关支持属性看337页。

    • 相同的 writing-mode 属性值并不会累加。
    • 垂直流有：
      • 水平方向也能 margin 合并。
      • 普通块元素可以使用 margin:auto 实现垂直居中。
      • 可以使用 text-align:center 实现图片垂直居中。
      • 可以使用 text-indent 实现文字下沉效果。
      • 可以实现全兼容的 icon fonts 图标的旋转效果。
      • 充分利用高度的高度自适应布局。

  9. writing-mode、direction 和 unicode-bidi 是 CSS 世界中三大可以改变文本布局
    流向的属性，其中 direction 和 unicode-bidi 属于近亲，经常一起使用，也是仅有的两个不受 CSS3 的 all 属性影响的 CSS 属性，基本上就是和内联元素一起使用。
